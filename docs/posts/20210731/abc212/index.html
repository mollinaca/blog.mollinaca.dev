A - Alloy

問題: A - Alloy

整数AとBが入力される。
A == 0 なら “Sliver”, B == 0 なら “Gold”, それ以外なら “Alloy” を出力する。

A,B = map(int,input().split())

if 0 < A and 0 < B:
    print ("Alloy")
elif B == 0:
    print ("Gold")

elif A == 0:
    print ("Silver")
else:
    pass

問題文通り実装するだけ。最後の else:pass はいらないな。

B - Weak Password

問題: B - Weak Password

文字列として数字4桁が入力される。
以下の条件を満たす場合、 “Weak” と出力し、それ以外の場合 “Strong” と出力する。

  • 4桁とも同じ数字
  • 1≤i≤3 をみたす任意の整数 i について、Xi+1 が、Xi の次の数字である。ただし9の次は0である。
p = input()

if p[0] == p[1] == p[2] == p[3]:
    ans = "Weak"
else:
    if (int(p[0])+1 == int(p[1]) or int(p[0])-9 == int(p[1])) and (int(p[1])+1 == int(p[2]) or int(p[1])-9 == int(p[2])) and (int(p[2])+1 == int(p[3]) or int(p[2])-9 == int(p[3])):
        ans = "Weak"
    else:
        ans = "Strong"

print (ans)

条件と確認ががややめんどくさいが、書くだけ。
もっとスマートな書き方あると思う。

C - Min Difference

問題: C - Min Difference

2つの数列A,Bが与えられるので、それぞれの中の任意の数字一つずつの差分が最も小さくなるものを出力する。

import bisect
N,M = map(int,input().split())
A = sorted(list(set(list(map(int,input().split())))))
B = sorted(list(set(list(map(int,input().split())))))
lenA = len(A)
lenB = len(B)

ans = float('inf')
if lenA <= lenB:
    for a in A:
        i = bisect.bisect_right(B,a)
        if i == 0:
            ans = min(ans, abs(a-B[i]))
        elif i == lenB:
            ans = min(ans, abs(a-B[i-1]))
        else:
            ans = min(ans, abs(a-B[i]), abs(a-B[i-1]))

elif lenB < lenA:
    for b in B:
        i = bisect.bisect_right(A,b)
        if i == 0:
            ans = min(ans, abs(b-A[i]))
        elif i == lenA:
            ans = min(ans, abs(b-A[i-1]))
        else:
            ans = min(ans, abs(b-A[i]), abs(b-A[i-1]))

print (ans)

素直に全部調べるとTLEする（はず）。
今回は、A、Bのうち短いほうを、長い方にたいして二分探索で最も近いものを確認し、それぞれと比較することで最小値を求めた。
A と B の長さ N,M が与えられるのでそれをつかえば単純にどっちが短いかわかるかなって思ったけど、サンプル3で同じ数字が複数回入るパターンがあり、
それだと長さの比較に N と M は使えないなぁと思ったので、重複の排除のために一度 set にして、改めて list にして長さを比較した。
ただしこれをやってもワーストケース（つまりAの長さとBの長さが等しい）パターンはあるので、これは本質的にあんまり意味ない。
たぶんこの二分探索ができてれば良い問題のはず。
