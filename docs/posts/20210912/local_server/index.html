自宅用ローカルサーバ構築時のメモ

環境

  • H/W: Raspberry Pi 4 Model B Rev 1.4
  • OS: 20.04.3 LTS (Focal Fossa)

作業内容・手順

OS用意

RaspberryPi に刺す SSD カードに OS イメージを用意する。 RaspberryPi Imager を使う。 メジャーなOS は Imager の GUI 上で DL できる。今回はここから Ubuntu 20.04.3 LTS を選択する。

OS を選択し、SSD カードに書き込む。

起動

RasberryPi に SSD カード、電源ケーブル、LANケーブルを刺して電源を入れる。無事起動したら ubuntu ユーザでログインする。 初期パスワードは ubuntu で、その場でパスワード変更を要求されるので対応する。

設定作業メモ

以下作業内容とコマンドのメモ。

apt-update && apt-upgrade

なにはともあれ。

# apt-update && apt-update

キーボードレイアウトを変更する

デフォルトだと英語キーボード設定になっているので、日本語キーボードに修正する。

# dpkg-reconfigure keyboard-configuration
 ->「Generic 105-key (Intl) PC」を選択
 -> 「Japanese」を選択
 ->「Japanese」を選択
 -> 「The default for the keyboard layout」を選択
 -> 「No compose key」を選択
# shutdown -r now

再起動後反映を確認する。

作業用ユーザ作成

デフォルトユーザをそのまま使い続けるのはセキュリティリスクになるので、別途作業用ユーザを作成する。ここでは自分のアカウントを sho とする。

# groupadd sho
# useradd -u 9001 -g 1001 -s /bin/bash -m -d /home/sho sho

作業用ユーザに root 権限 NOPASSWD を付与する。
※この設定はあくまでローカルサーバでの使用する。インターネット環境に公開するサーバでは設定しない。

# vi /etc/sudoers
===
# User privilege specification
root    ALL=(ALL:ALL) ALL
sho     ALL=(ALL) NOPASSWD: ALL
===

※ visudo を実行するとデフォルトで nano エディタで起動してくるので、 /etc/sudoers を直接編集する。のちほどデフォルトのエディタも変更する。

作成したユーザに公開鍵を設定し、sshログインができることを確認する。

デフォルトユーザ無効化

削除してもよいけど、なんとなく削除せんでいいもんは削除したくないので、ログインできなくしておく。いろいろ方法あるけど、自分は以下の方法で普段対応している。

# usermod -s /sbin/nologin ubuntu

これにより、ログインシェルがログイン不可の /sbin/nologin になる。
この設定にした場合、ubuntu ユーザへのログイン施行と認証の成功がログに残るので、外部から攻撃をうけているかどうかが確認しやすい。

ホスト名変更

# hostnamectl set-hostname raspberry1

TimeZone 変更

# timedatectl set-timezone Asia/Tokyo

IPアドレスを固定にする

今回は有線LANで接続しているので、 eth0 にDHCPサーバからデフォルトのIPアドレスが配布されている。任意の固定IPアドレスに変更する。

# vi /etc/netplan/99_config.yaml
===
network:
  version: 2
  renderer: networkd
  ethernets:
    eth0:
      dhcp4: false
      dhcp6: false
      addresses: [192.168.100.202/24]
      gateway4: 192.168.100.1
      nameservers:
        addresses: [8.8.8.8, 8.8.4.4]
===
# netplan apply try --timeout 10

今回は、 192.168.100.202 というアドレスに設定した。
netplay apply は設定ファイルがおかしかったりして失敗するとそのままIPアドレスを失ったりしてサーバへの接続ができなくなる恐れがあるので、実行時に try オプションをつける。これをつけると、失敗した場合に自動でロールバックしてくれる。

ログ出力を抑制する

RaspberryPi の場合、 OS 領域は SD カードにインストールされている。あまりログをガリガリ出力すると、SD カードの寿命を縮めてしまうので、延命のためになるべくログを出力しなくする。

・ /etc/rsyslog.d/20-ufw.conf → コメントアウト
・ /etc/rsyslog.d/21-cloudinit.conf → コメントアウト
・ /etc/rsyslog.d/50-default.conf → コメントアウト

swap を無効化する

同様に、SD カードを swap 領域として使用しないようにするために、 swap を無効化する。

# swapoff --all

ここから先はお好みで

docker インストール

公式な手順はこちら

# curl -sSL https://get.docker.com | sh

docker-compose は pip からインストールする。

# apt-get install pip3
# pip3 install docker-compose

pip はデフォルトで入ってなくて、pip入れると余計なものがたくさんついてくるので、SD カードのことを考えたらここも docker-compose を単体でいれたほうがよいかもしれない。

なんにせよ、これで docker と docker-compose は動くようになった。
なお、RaspberryPi は ARM なので、Docker の Intel(x86_64) イメージはそのままでは動かない。 ARM に対応したイメージを持ってくるか、さもなくば自作する必要がある。

温度チェックツール

便利ツール。

# apt-get install libraspberrypi-bin
# vcgencmd measure_temp
temp=61.3'C

NETDATA

NETDATA はサーバのメトリクスを取得しリアルタイムに WEB GUI で表示してくれるツール。リッチな UI でグリグリ動くため、とりあえずいれておくと満足度が高い。

手順 pi-hole-raspberry-pi および Install Netdata with Docker

NETDATA は Docker イメージも公開されており、 ARM に対応したイメージもあるのでそれを使う。

version: '3'
services:
  netdata:
    image: netdata/netdata
    container_name: netdata
    hostname: example.com # set to fqdn of host
    ports:
      - 19999:19999
    restart: unless-stopped
    cap_add:
      - SYS_PTRACE
    security_opt:
      - apparmor:unconfined
    volumes:
      - netdataconfig:/etc/netdata
      - netdatalib:/var/lib/netdata
      - netdatacache:/var/cache/netdata
      - /etc/passwd:/host/etc/passwd:ro
      - /etc/group:/host/etc/group:ro
      - /proc:/host/proc:ro
      - /sys:/host/sys:ro
      - /etc/os-release:/host/etc/os-release:ro

volumes:
  netdataconfig:
  netdatalib:
  netdatacache:

# docker-compose up -d
 -> NETDATA の Docker コンテナが起動し、8080 で Listen する

# docker ps -a | grep netdata
ee296ec7ea8d   netdata/netdata   "/usr/sbin/run.sh"       25 hours ago   Up 25 hours (healthy)   0.0.0.0:19999->19999/tcp, :::19999->19999/tcp   netdata

上記までで、一通り動いてローカルで使える ubuntu サーバを RaspberryPi 上に構築できた。
